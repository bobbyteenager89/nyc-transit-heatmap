## Consolidation candidates (for Claude Design)

### Tier 1 — extract now
1. **Button/CTA system** — unify ModeCard CTA, Chip, ShareSheet/ShareButton, PlayButton, snap buttons → `Button` primitive w/ variants (primary, secondary, ghost, icon)
2. **Slider primitive** — merge TimeSlider + FairnessSlider under one component with `theme` prop
3. **Callout card** — BestNeighborhood + SurpriseInsight + MeetupSummary share a pattern (tinted border + icon + heading + body + optional action)
4. **Color + shape tokens** — move MTA palette, time-ramp, radii, shadows into `tokens.ts`

### Tier 2 — medium complexity
1. **Modal** — inline "How it works" dialogs on Explore + Find should be a primitive
2. **Dropdown/Select** — Mapbox autocomplete + Compare "Add neighborhood" have different styles
3. **Loading state** — spinner + skeleton primitives
4. **Grid layout** — responsive 1–3 col pattern appears in Compare + potential results grids

### Tier 3 — theme isolation
1. **Wizard brutalist theme** — currently pink/red inline everywhere. Move to a CSS-variable `data-theme="wizard"` scope on WizardShell so Claude Design can swap it cleanly.
2. **ModeLegend vs ModeToggles** — same underlying toggle, different shell. Merge with optional `icons` + `columns` props.

### Inconsistencies flagged
- `text-[10px] uppercase tracking-widest` — ~15 duplicate uses. Need a `.label-micro` utility.
- `min-h-[44px]` hardcoded — needs `--touch-target: 44px`.
- `tabular-nums` applied inconsistently; should be default on all numeric displays.
- `transition-all` was purged in S27; don't reintroduce.
